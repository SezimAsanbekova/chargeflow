# Тестирование Email уведомлений

## ✅ Что реализовано

При входе и регистрации пользователю отправляется email уведомление:
- **При регистрации:** "Добро пожаловать в ChargeFlow!"
- **При входе:** "Вход в аккаунт ChargeFlow"

## 🧪 Как протестировать

### 1. Регистрация нового пользователя

```bash
# Откройте терминал где запущен сервер
# Вы увидите логи email

# В браузере:
1. Откройте http://localhost:3000/auth/signin
2. Нажмите "Нет аккаунта? Зарегистрируйтесь"
3. Введите:
   - Имя: Тест
   - Email: test@example.com
   - Пароль: test123
4. Нажмите "Зарегистрироваться"
```

**Ожидаемый результат в терминале:**
```
📧 Email отправлен:
To: test@example.com
Subject: Добро пожаловать в ChargeFlow!
HTML: <!DOCTYPE html>...
```

### 2. Вход существующего пользователя

```bash
# В браузере:
1. Выйдите из аккаунта (если вошли)
2. Откройте http://localhost:3000/auth/signin
3. Введите email и пароль
4. Нажмите "Войти"
```

**Ожидаемый результат в терминале:**
```
📧 Email отправлен:
To: test@example.com
Subject: Вход в аккаунт ChargeFlow
HTML: <!DOCTYPE html>...
```

### 3. Вход через Google

```bash
# В браузере:
1. Выйдите из аккаунта
2. Откройте http://localhost:3000/auth/signin
3. Нажмите "Войти через Google"
4. Выберите аккаунт (уже зарегистрированный)
```

**Ожидаемый результат в терминале:**
```
📧 Email отправлен:
To: your-google@email.com
Subject: Вход в аккаунт ChargeFlow
HTML: <!DOCTYPE html>...
```

## ⚙️ Режимы работы

### Development (текущий):
- ✅ Email логируются в консоль
- ✅ Не отправляются реально
- ✅ Можно видеть HTML код
- ✅ Быстро и удобно для тестирования

### Production (после настройки):
- ✅ Email отправляются реально
- ✅ Через Resend, SendGrid или другой сервис
- ✅ Пользователи получают письма
- ✅ Требует API ключ

## 🚀 Следующие шаги

### Для production:

1. **Выберите email сервис:**
   - Resend (рекомендуется) - https://resend.com
   - SendGrid - https://sendgrid.com
   - Mailgun - https://mailgun.com

2. **Получите API ключ**

3. **Обновите `lib/email.ts`** (см. EMAIL_NOTIFICATIONS.md)

4. **Добавьте в `.env`:**
   ```
   RESEND_API_KEY="re_..."
   ```

5. **Протестируйте в production**
