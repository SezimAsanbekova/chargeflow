# Перенаправление на профиль после входа

## ✅ Изменено

После регистрации или входа пользователь теперь перенаправляется на `/profile` вместо главной страницы.

## Что изменилось

### 1. Страница входа/регистрации (`app/auth/signin/page.tsx`)

#### Было:
```typescript
const callbackUrl = searchParams.get('callbackUrl') || '/';

// После входа
router.push(callbackUrl);

// После регистрации
router.push(callbackUrl);
```

#### Стало:
```typescript
const callbackUrl = searchParams.get('callbackUrl') || '/profile';

// После входа
router.push(callbackUrl);

// После регистрации
router.push('/profile'); // Всегда на профиль
```

## Поведение

### 1. Обычный вход (email + пароль)
```
Пользователь → /auth/signin
           ↓
    Вводит email/пароль
           ↓
    Нажимает "Войти"
           ↓
    Перенаправление на /profile
```

### 2. Регистрация (email + пароль)
```
Пользователь → /auth/signin
           ↓
    Переключается на "Регистрация"
           ↓
    Вводит данные
           ↓
    Нажимает "Зарегистрироваться"
           ↓
    Автоматический вход
           ↓
    Перенаправление на /profile
```

### 3. Вход через Google
```
Пользователь → /auth/signin
           ↓
    Нажимает "Войти через Google"
           ↓
    Выбирает аккаунт Google
           ↓
    Перенаправление на /profile
```

### 4. Защищенные страницы
Если пользователь пытается открыть защищенную страницу без авторизации:
```
Пользователь → /vehicles (защищена)
           ↓
    Редирект на /auth/signin?callbackUrl=/vehicles
           ↓
    Вход
           ↓
    Перенаправление на /vehicles (оригинальная страница)
```

## Примеры

### Пример 1: Прямой вход

```
2. Нажать "Нет аккаунта? Зарегистрируйтесь"

4. Нажать "Зарегистрироваться"
5. Результат: http://localhost:3000/profile
```

### Пример 3: Вход через Google
```
1. Открыть http://localhost:3000/auth/signin
2. Нажать "Войти через Google"
3. Выбрать аккаунт
4. Результат: http://localhost:3000/profile
```

### Пример 4: Доступ к защищенной странице
```
1. Открыть http://localhost:3000/vehicles (без авторизации)
2. Редирект на /auth/signin?callbackUrl=/vehicles
3. Войти
4. Результат: http://localhost:3000/vehicles
```

## Логика callbackUrl

### По умолчанию:
- Если `callbackUrl` не указан → `/profile`
- Если `callbackUrl` указан → используется указанный URL

### Примеры:
```typescript
// Прямой вход
/auth/signin → /profile

// Из защищенной страницы
/auth/signin?callbackUrl=/vehicles → /vehicles

// Из защищенной страницы (профиль)
/auth/signin?callbackUrl=/profile → /profile

// Регистрация (всегда профиль)
/auth/signin (регистрация) → /profile
```

## Возможные проблемы

### Бесконечный редирект
**Причина:** Профиль защищен, но пользователь не авторизован

**Решение:** Убедитесь, что:
- Сессия создается после входа
- Cookie устанавливается
- Proxy.ts правильно проверяет авторизацию

### Редирект на главную вместо профиля
**Причина:** Кеш браузера

**Решение:**
1. Очистите кеш (Ctrl+Shift+R)
2. Перезапустите сервер
3. Попробуйте снова

### callbackUrl игнорируется
**Причина:** Ошибка в логике

**Решение:** Проверьте логи в DevTools → Network
