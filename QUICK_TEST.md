# Быстрое тестирование Google OAuth

## ✅ Что исправлено

Теперь при входе через Google:
1. Автоматически создается пользователь в БД
2. Создается баланс (0.00)
3. Пользователь перенаправляется на главную страницу
4. Показывается UserMenu с именем и аватаром

## 🧪 Как протестировать

### Шаг 1: Очистите cookies
**Chrome/Edge:**
- F12 → Application → Cookies → localhost:3000 → Clear all

**Firefox:**
- F12 → Storage → Cookies → localhost:3000 → Delete all

**Safari:**
- Develop → Empty Caches

### Шаг 2: Обновите страницу
```
Ctrl+Shift+R (Windows/Linux)
Cmd+Shift+R (Mac)
```

### Шаг 3: Нажмите "Войти"
На главной странице в правом верхнем углу

### Шаг 4: Выберите "Войти через Google"
Зеленая кнопка с иконкой Google

### Шаг 5: Выберите Google аккаунт
Выберите аккаунт, которым хотите войти

### Шаг 6: Нажмите "Продолжить"
Разрешите доступ к email и профилю

### Шаг 7: Проверьте результат

**✅ Должно произойти:**
- Перенаправление на главную страницу (http://localhost:3000)
- В правом верхнем углу появится UserMenu
- Отобразится ваше имя из Google
- Отобразится аватар из Google
- Баланс: 0.00 сом

**❌ Если не работает:**
1. Откройте DevTools (F12)
2. Перейдите на вкладку Console
3. Скопируйте ошибки
4. Проверьте логи сервера в терминале

## 🔍 Проверка в БД

### Откройте Prisma Studio:
```bash
npm run db:studio
```

### Проверьте таблицы:

**users:**
- Должен быть новый пользователь
- email: ваш Google email
- name: ваше имя из Google
- image: URL аватара
- emailVerified: текущая дата
- role: user
- status: active
- passwordHash: null

**accounts:**
- Должна быть запись
- provider: google
- type: oauth
- userId: ID созданного пользователя

**user_balance:**
- Должна быть запись
- userId: ID пользователя
- balance: 0

## 🐛 Возможные проблемы

### 1. Остается на странице входа
**Причина:** Ошибка при создании пользователя

**Решение:**
```bash
# Проверьте логи
tail -f .next/dev/logs/next-development.log

# Перезапустите сервер
# Ctrl+C в терминале, затем:
npm run dev
```

### 2. Ошибка "Configuration"
**Причина:** Неверные Google credentials

**Решение:**
```bash
# Проверьте .env
cat .env | grep GOOGLE

# Должно быть:
GOOGLE_CLIENT_ID="ваш_реальный_client_id"
GOOGLE_CLIENT_SECRET="ваш_реальный_secret"
```

### 3. Ошибка "Redirect URI mismatch"
**Причина:** Неверный redirect URI в Google Console

**Решение:**
1. Откройте Google Cloud Console
2. APIs & Services → Credentials
3. Выберите ваш OAuth 2.0 Client
4. Authorized redirect URIs должен содержать:
   ```
   http://localhost:3000/api/auth/callback/google
   ```

### 4. UserMenu не появляется
**Причина:** Кеш браузера

**Решение:**
1. Очистите cookies (см. Шаг 1)
2. Обновите страницу (Ctrl+Shift+R)
3. Проверьте DevTools → Network → session
4. Должен быть запрос к `/api/auth/session` с ответом 200
