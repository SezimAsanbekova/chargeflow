# Техническое задание (ТЗ)

**Мобильное приложение для управления зарядными станциями для электромобилей**

**Проект:** Система управления зарядной инфраструктурой и взаимодействия с пользователями  
**Формат:** Мобильное приложение (Android / iOS) + Backend (Django) + БД (PostgreSQL)  
**Роли:** Пользователь (владелец электромобиля), Администратор сети зарядных станций  
**Количество станций на старте:** от 5 до 20 (масштабируемо)

---

## 1. Общая цель проекта

Разработать мобильное приложение, которое позволяет:

- находить зарядные станции с учетом параметров автомобиля;
- бронировать зарядные точки;
- управлять зарядной сессией;
- оплачивать зарядку через приложение;
- хранить и управлять автомобилями пользователя;
- автоматически подбирать подходящие станции;
- администратору — управлять станциями, пользователями и получать аналитические отчёты.

---

## 2. Цели разработки

1. Автоматизировать поиск и бронирование зарядных станций
2. Обеспечить удалённый запуск и остановку зарядки
3. Интегрировать безопасную онлайн-оплату
4. Реализовать персонализированный подбор станций под автомобиль
5. Предоставить администратору прозрачную аналитику
6. Обеспечить масштабируемость системы

---

## 3. Тип системы

- **Мобильное приложение** (iOS / Android)

## 5. Функциональные требования

### 5.1 Роль «Пользователь»

#### 5.1.1 Регистрация и авторизация

- регистрация по номеру телефона (SMS‑код)
- вход по паролю или коду
- восстановление пароля
- подтверждение номера

#### 5.1.2 Управление автомобилями 🚗

Пользователь может добавить один или несколько автомобилей.

**Для каждого автомобиля указываются:**

- марка
- модель
- год выпуска
- тип коннектора (CCS2, CHAdeMO, Type 2, GB/T)
- максимальная мощность зарядки (кВт)
- ёмкость батареи (кВт·ч)
- уровень заряда (опционально)

**Функции:**

- выбор активного автомобиля
- редактирование / удаление
- хранение нескольких авто

#### 5.1.3 Поиск и фильтрация станций 🔍

- карта с геолокацией
- список станций

**Фильтры:**

- тип станции
- тип коннектора
- мощность (диапазон)
- доступность

**📌 Дополнительно (персонализация):**

Система должна:

- автоматически учитывать выбранный автомобиль
- показывать только совместимые станции
- сортировать станции по эффективности зарядки
- приоритет отображения станций с подходящей мощностью зарядки

#### 5.1.4 Просмотр станции

- адрес и координаты
- фото
- список коннекторов
- мощность
- цена за кВт·ч
- статус
- график работы
- уровень совместимости с автомобилем
- статус каждого коннектора (свободен / занят / неисправен)

#### 5.1.5 Бронирование

- выбор даты и времени
- подтверждение брони
- отмена брони
- история бронирований

**Дополнительно:**

- при бронировании взимается депозит в размере 100 сом
- депозит списывается с банковской карты или внутреннего баланса
- пользователь может отменить бронирование **не позднее чем за 1 час до начала**
- при своевременной отмене депозит возвращается
- если до начала бронирования осталось менее 1 часа:
  - отмена невозможна
  - депозит не возвращается
- после наступления времени бронирования станция ожидает пользователя **15 минут**
- если пользователь не начал зарядку в течение 15 минут:
  - бронирование автоматически отменяется
  - станция становится доступной
  - депозит не возвращается

#### 5.1.6 Зарядная сессия ⚡

- запуск через QR‑код
- запуск через приложение
- отображение в реальном времени:
  - энергия (кВт·ч)
  - время
  - стоимость
- остановка зарядки

**Дополнительно:**

- восстановление при обрыве связи
- защита от двойного запуска
- автоматический расчёт стоимости в процессе зарядки

#### 5.1.7 Оплата 💳

- привязка банковской карты (через платежный шлюз)
- оплата:
  - картой
  - внутренним балансом
- автоматическое списание после сессии
- просмотр чеков
- история платежей
- возврат средств (например, при отмене бронирования)

#### 5.1.8 История

- список зарядных сессий
- детали сессии
- история платежей

#### 5.1.9 Уведомления 🔔

Система должна отправлять:

- напоминание о бронировании
- уведомление о начале/окончании зарядки
- ошибка зарядки
- недостаточно средств

### 5.2 Роль «Администратор»

#### 5.2.1 Управление станциями

- добавление / редактирование / удаление
- управление параметрами:
  - адрес
  - координаты
  - коннекторы
  - мощность
  - цена
  - статус
  - график работы

#### 5.2.2 Управление пользователями

- список пользователей
- блокировка / разблокировка
- просмотр истории

#### 5.2.3 Зарядные сессии

- просмотр всех сессий
- фильтрация:
  - по станции
  - по пользователю
  - по дате
  - по статусу

#### 5.2.4 Отчёты 📊

**За день:**

- количество сессий
- энергия
- выручка
- средний чек

**За неделю:**

- динамика по дням
- загруженность
- доля бронирований

**За месяц:**

- топ станций
- топ пользователей
- средняя длительность
- пиковые часы

**За год:**

- выручка
- рост
- сезонность
- энергия

**Дополнительно:**

- экспорт (Excel / PDF)
- произвольный период
- графики

#### 5.2.5 Логи

- действия администратора
- изменения данных
- история операций

## 7. Сценарии работы

**Пользователь:**

1. Открывает приложение
2. Выбирает автомобиль
3. Получает список подходящих станций
4. Бронирует
5. Приезжает и сканирует QR
6. Зарядка начинается
7. Оплачивает
8. Получает чек

**Администратор:**

1. Входит в систему
2. Открывает отчёты
3. Анализирует данные
4. Экспортирует отчёт
