# 🔐 Функции безопасности ChargeFlow

## ✅ Реализованные функции безопасности

### 1. Требования к безопасному паролю

При регистрации пароль должен соответствовать следующим требованиям:

- ✅ **Минимум 8 символов**
- ✅ **Минимум 1 заглавная буква** (A-Z, А-Я)
- ✅ **Минимум 1 строчная буква** (a-z, а-я)
- ✅ **Минимум 1 цифра** (0-9)
- ✅ **Минимум 1 специальный символ** (!@#$%^&*()_+-=[]{}|;:,.<>?)

**Примеры безопасных паролей:**
- `MyPass123!`
- `Secure@2026`
- `Зарядка#123`

**Примеры небезопасных паролей:**
- `password` ❌ (нет заглавных букв, цифр, символов)
- `12345678` ❌ (нет букв и символов)
- `Password` ❌ (нет цифр и символов)
- `Pass1!` ❌ (меньше 8 символов)

---

### 2. Защита от брутфорса (перебора паролей)

**Механизм защиты:**

1. **Отслеживание попыток входа**
   - Каждая неудачная попытка входа увеличивает счетчик
   - Пользователь видит сколько попыток осталось

2. **Автоматическая блокировка**
   - После **5 неудачных попыток** аккаунт блокируется на **1 час**
   - Блокировка автоматическая, не требует вмешательства администратора

3. **Сброс счетчика**
   - При успешном входе счетчик сбрасывается на 0
   - Блокировка снимается автоматически через 1 час

**Сообщения пользователю:**

- После 1-й неудачной попытки: `"Неверный пароль. Осталось попыток: 4"`
- После 2-й неудачной попытки: `"Неверный пароль. Осталось попыток: 3"`
- После 5-й неудачной попытки: `"Слишком много неудачных попыток входа. Аккаунт заблокирован на 1 час"`
- При попытке входа в заблокированный аккаунт: `"Аккаунт заблокирован. Попробуйте через 45 минут"`

## 📊 Технические детали

### База данных

Добавлены поля в таблицу `users`:

```sql
login_attempts INT DEFAULT 0
locked_until TIMESTAMP NULL
```

### API изменения

**POST /api/auth/register**
- Добавлена валидация пароля
- Возвращает детальные ошибки при несоответствии требованиям

**Credentials Provider (NextAuth)**
- Проверка блокировки перед проверкой пароля
- Увеличение счетчика при неудачной попытке
- Автоматическая блокировка после 5 попыток
- Сброс счетчика при успешном входе

## 🧪 Тестирование

### Тест 1: Валидация пароля при регистрации

1. Попробуйте зарегистрироваться с паролем `12345678`
   - ❌ Должна появиться ошибка с требованиями

2. Попробуйте зарегистрироваться с паролем `MySecure123!`
   - ✅ Регистрация должна пройти успешно

### Тест 2: Защита от брутфорса

1. Зарегистрируйте тестовый аккаунт
2. Выйдите из аккаунта
3. Попробуйте войти с **неправильным паролем** 5 раз
4. После 5-й попытки аккаунт должен быть заблокирован
5. Попробуйте войти с **правильным паролем** - должна быть ошибка блокировки
6. Подождите 1 час (или измените `locked_until` в БД для быстрого теста)
7. Войдите с правильным паролем - вход должен быть успешным

### Быстрый тест блокировки (для разработки)

```sql
-- Вручную заблокировать пользователя на 5 минут
UPDATE users 
SET login_attempts = 5, 
    locked_until = NOW() + INTERVAL '5 minutes'
WHERE email = 'test@example.com';

-- Проверить статус блокировки
SELECT email, login_attempts, locked_until 
FROM users 
WHERE email = 'test@example.com';

-- Разблокировать пользователя
UPDATE users 
SET login_attempts = 0, 
    locked_until = NULL
WHERE email = 'test@example.com';
```
