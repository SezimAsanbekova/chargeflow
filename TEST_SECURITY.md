3. **Ожидаемый результат:** Успешная регистрация, переход в профиль

## Тест 2: Защита от брутфорса 🔒

### Подготовка
1. Зарегистрируйте тестовый аккаунт:
   - Email: `bruteforce@test.com`
   - Пароль: `TestPass123!`
2. Выйдите из аккаунта

### Тест блокировки
1. Попробуйте войти с **неправильным паролем**: `WrongPass123!`
2. **После 1-й попытки:** "Неверный пароль. Осталось попыток: 4"
3. **После 2-й попытки:** "Неверный пароль. Осталось попыток: 3"
4. **После 3-й попытки:** "Неверный пароль. Осталось попыток: 2"
5. **После 4-й попытки:** "Неверный пароль. Осталось попыток: 1"
6. **После 5-й попытки:** "Слишком много неудачных попыток входа. Аккаунт заблокирован на 1 час"

### Проверка блокировки
1. Попробуйте войти с **правильным паролем**: `TestPass123!`
2. **Ожидаемый результат:** "Аккаунт заблокирован. Попробуйте через X минут"

### Быстрая разблокировка (для теста)
```bash
# Подключитесь к PostgreSQL
psql -U postgres -d chargeflow

# Разблокируйте аккаунт
UPDATE users 
SET login_attempts = 0, locked_until = NULL 
WHERE email = 'bruteforce@test.com';

# Выйдите
\q
```

### Проверка разблокировки
1. Войдите с правильным паролем: `TestPass123!`
2. **Ожидаемый результат:** Успешный вход

## Тест 3: Email уведомления 📧

### При регистрации
1. Зарегистрируйте новый аккаунт
2. Проверьте терминал - должно быть:
   ```
   ✅ Email успешно отправлен
   To: your@email.com
   Subject: Добро пожаловать в ChargeFlow!
   ```

### При входе
1. Войдите в существующий аккаунт
2. Проверьте терминал - должно быть:
   ```
   ✅ Email успешно отправлен
   To: your@email.com
   Subject: Вход в аккаунт ChargeFlow
   ```

## 🐛 Если что-то не работает

### Проблема: Пароль не проходит валидацию
**Решение:** Убедитесь, что пароль содержит:
- Минимум 8 символов
- Заглавные буквы (A-Z)
- Строчные буквы (a-z)
- Цифры (0-9)
- Спецсимволы (!@#$%^&*)

### Проблема: Блокировка не работает
**Решение:** 
1. Проверьте, что миграция применена: `npx prisma migrate status`
2. Проверьте поля в БД: `SELECT login_attempts, locked_until FROM users;`

### Проблема: Email не отправляются
**Решение:** 
1. Проверьте настройки в `.env` файле
2. Убедитесь, что EMAIL_USER и EMAIL_PASS заполнены
3. В development режиме письма логируются в консоль, а не отправляются реально
