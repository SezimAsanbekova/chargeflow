## 🏗️ Архитектура

### Процесс входа

```
1. Пользователь → /admin/signin
   ↓
2. Ввод email + Telegram ID
   ↓
3. POST /api/admin/send-code
   ↓
4. Проверка в БД:
   - Существует ли пользователь?
   - Роль = admin?
   - Telegram ID совпадает?
   ↓
5. Генерация 6-значного кода
   ↓
6. Сохранение в email_verification_codes
   ↓
7. Отправка через Telegram Bot API
   ↓
8. Пользователь → /admin/verify-code
   ↓
9. Ввод кода
   ↓
10. POST /api/admin/verify-code
    ↓
11. Проверка кода в БД
    ↓
12. Генерация JWT токена
    ↓
13. Установка cookie 'admin-token'
    ↓
14. Redirect → /admin/dashboard
```

### Защита маршрутов

```
Middleware проверяет:

/admin/* (кроме signin, verify-code)
  ↓
Есть cookie 'admin-token'?
  ↓ Нет
  Redirect → /admin/signin
  ↓ Да
Валидный JWT?
  ↓ Нет
  Удалить cookie → Redirect → /admin/signin
  ↓ Да
Роль = admin?
  ↓ Нет
  Redirect → /admin/signin
  ↓ Да
  Доступ разрешен ✅

/profile, /vehicles, /booking, /charging
  ↓
Есть cookie 'admin-token'?
  ↓ Да
  Redirect → /admin/dashboard
  ↓ Нет
Есть NextAuth токен?
  ↓ Нет
  Redirect → /auth/signin
  ↓ Да
  Доступ разрешен ✅
```

## 🔧 Настройка

### 1. Установка зависимостей

```bash
npm install
```

Уже установлено:
- `jsonwebtoken` - для JWT токенов
- `@types/jsonwebtoken` - типы для TypeScript

### ⚠️ Важно для Next.js 16

В Next.js 16 файл `middleware.ts` устарел и заменен на `proxy.ts`. Вся логика защиты маршрутов реализована в `proxy.ts`.

### 2. Переменные окружения

Добавьте в `.env`:

```env
JWT_SECRET="ваш_секретный_ключ_минимум_32_символа"
```

### 3. Создание Telegram бота

1. [@BotFather](https://t.me/BotFather) → `/newbot`
2. Сохраните токен

### 4. Получение Telegram ID

1. [@userinfobot](https://t.me/userinfobot)
2. Сохраните ID

### 5. Настройка базы данных

#### Вариант 1: SQL

```sql
-- Создать администратора
INSERT INTO users (id, email, name, role, status, email_verified, created_at, updated_at)
VALUES (gen_random_uuid(), 'admin@example.com', 'Администратор', 'admin', 'active', NOW(), NOW(), NOW());

-- Создать баланс
INSERT INTO user_balance (user_id, balance, updated_at)
SELECT id, 0, NOW() FROM users WHERE email = 'admin@example.com';

-- Добавить настройки
INSERT INTO settings (id, key, value, updated_at)
VALUES 
  (gen_random_uuid(), 'ADMIN_TELEGRAM_BOT_TOKEN', 'ВАШ_ТОКЕН', NOW()),
  (gen_random_uuid(), 'ADMIN_TELEGRAM_USER_ID', 'ВАШ_ID', NOW())
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;
```

#### Вариант 2: Seed скрипт

```bash
# 1. Отредактируйте prisma/seed-admin.ts
# 2. Замените YOUR_TELEGRAM_BOT_TOKEN_HERE и YOUR_TELEGRAM_USER_ID_HERE
# 3. Запустите:
npm run db:seed:admin
```

### 6. Начните диалог с ботом

Найдите бота в Telegram и отправьте `/start`

### 7. Войдите

Откройте `http://localhost:3000/admin/signin`

## 📝 Примечания

### Использование существующих таблиц

Система использует существующие таблицы базы данных:

- `users` - для хранения администраторов (поле `role = 'admin'`)
- `settings` - для хранения Telegram настроек
- `email_verification_codes` - для хранения кодов верификации

Не требуется создание новых таблиц или миграций.

### Совместимость с пользовательской системой

Админская система полностью изолирована от пользовательской:

- Разные токены (admin-token vs next-auth.session-token)
- Разные маршруты (/admin/* vs /profile/*, /vehicles/*, etc.)
- Разные API endpoints (/api/admin/* vs /api/user/*, /api/auth/*)
- Разные цветовые схемы (янтарный vs зеленый)
