# Исправление сохранения телефона

## Проблема
Ошибка "Не удалось сохранить номер телефона" при попытке сохранить номер.

## Причина
После добавления поля `phone` в схему Prisma и применения миграции, Prisma Client не был обновлен. Поэтому TypeScript типы не содержали новое поле.

## Решение

### 1. Сгенерирован Prisma Client
```bash
npx prisma generate
```

### 2. Очищен кеш Next.js
```bash
rm -rf .next
```

### 3. Перезапущен сервер
```bash
npm run dev
```

## Как работает

### Сохранение телефона:
1. Пользователь открывает `/profile/phone`
2. Вводит номер (автоформатирование)
3. Нажимает "Сохранить"
4. Отправляется PATCH запрос к `/api/user/profile`
5. Номер сохраняется в БД
6. Показывается сообщение "✓ Номер телефона сохранен"
7. Через 1.5 секунды редирект на `/profile`

### Без SMS верификации:
- ✅ Номер просто сохраняется в БД
- ✅ Никаких SMS не отправляется
- ✅ Сразу доступен для использования
- ✅ Можно изменить в любое время

## Тестирование

### 1. Откройте профиль
```
http://localhost:3000/profile
```

### 2. Нажмите на "Телефон"

### 3. Введите номер
Например: `996501979495`

Автоматически форматируется в: `+996 501 979 495`

### 4. Нажмите "Сохранить"

Должно появиться:
- ✓ Зеленая галочка
- Сообщение "✓ Номер телефона сохранен"
- Через 1.5 сек возврат на профиль

### 5. Проверьте профиль

В профиле должен отображаться сохраненный номер:
```
📱 Телефон
   +996 501 979 495
```

## API Endpoints

### GET /api/user/profile
Возвращает данные пользователя включая телефон:
```json
{
  "id": "uuid",
  "email": "user@example.com",
  "name": "Имя",
  "phone": "+996 501 979 495",
  "image": "url",
  "role": "user",
  "balance": 0,
  "createdAt": "2024-01-01T00:00:00.000Z"
}
```

### PATCH /api/user/profile
Обновляет телефон:
```json
{
  "phone": "+996 501 979 495"
}
```

Ответ:
```json
{
  "id": "uuid",
  "email": "user@example.com",
  "name": "Имя",
  "phone": "+996 501 979 495",
  "image": "url",
  "role": "user"
}
```

## Валидация

### Формат номера:
- Должен начинаться с `996`
- Длина: 12 цифр (996 + 9 цифр)
- Формат отображения: `+996 XXX XXX XXX`

### Примеры валидных номеров:
- ✅ `+996 501 979 495`
- ✅ `+996 555 123 456`
- ✅ `+996 700 999 888`

### Примеры невалидных номеров:
- ❌ `+7 777 123 4567` (не Кыргызстан)
- ❌ `+996 12 345` (слишком короткий)
- ❌ `996` (неполный номер)

## Возможные ошибки

### "Введите корректный номер телефона"
**Причина:** Номер не соответствует формату

**Решение:** Убедитесь, что:
- Номер начинается с 996
- Содержит 9 цифр после кода страны
- Итого 12 цифр

### "Не удалось сохранить номер телефона"
**Причина:** Ошибка сервера

**Решение:**
1. Проверьте логи в терминале
2. Убедитесь, что БД доступна
3. Проверьте, что миграция применена:
```bash
npx prisma migrate status
```

### Номер не отображается в профиле
**Причина:** Кеш браузера

**Решение:**
1. Обновите страницу (Ctrl+Shift+R)
2. Очистите cookies
3. Перезайдите в аккаунт
