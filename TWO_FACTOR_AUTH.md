# 🔐 Двухфакторная аутентификация (2FA) через Email

## ✅ Реализовано

### Процесс входа/регистрации с 2FA:

```
1. Пользователь вводит email + пароль
   ↓
2. Система проверяет данные
   ↓
3. Отправляется 6-значный код на email
   ↓
4. Пользователь вводит код на отдельной странице
   ↓
5. После правильного кода → вход в профиль
```

---

## 📋 Как это работает

### Шаг 1: Вход/Регистрация
**Страница:** `/auth/signin`

Пользователь вводит:
- Email
- Пароль
- (Имя - только при регистрации)

### Шаг 2: Отправка кода
**API:** `POST /api/auth/send-code`

Система:
1. ✅ Проверяет email и пароль
2. ✅ Проверяет блокировку аккаунта (5 попыток)
3. ✅ Генерирует 6-значный код (например: `123456`)
4. ✅ Сохраняет код в БД (срок действия: 10 минут)
5. ✅ Отправляет код на email
6. ✅ Переходит на страницу ввода кода

### Шаг 3: Ввод кода
**Страница:** `/auth/verify-code`

Пользователь:
- Видит 6 полей для ввода цифр
- Может вставить код из буфера обмена
- Автоматическая отправка при заполнении всех полей

### Шаг 4: Проверка кода
**API:** `POST /api/auth/verify-code`

Система:
1. ✅ Проверяет правильность кода
2. ✅ Проверяет срок действия (10 минут)
3. ✅ Помечает код как использованный
4. ✅ Сбрасывает счетчик неудачных попыток
5. ✅ Отправляет уведомление о входе
6. ✅ Выполняет вход через NextAuth
7. ✅ Переходит в профиль

## 🔒 Безопасность

### Защита кода:
- ✅ **Срок действия:** 10 минут
- ✅ **Одноразовый:** После использования код становится недействительным
- ✅ **Удаление старых:** При запросе нового кода старые удаляются
- ✅ **Только цифры:** 6-значный числовой код

### Защита от брутфорса:
- ✅ Проверка пароля **до** отправки кода
- ✅ Счетчик неудачных попыток (5 попыток)
- ✅ Блокировка на 1 час после 5 неудачных попыток
- ✅ Сброс счетчика при успешном входе

### Защита от спама:
- ✅ Таймер 60 секунд между отправками кода
- ✅ Удаление старых неиспользованных кодов

## 📊 База данных

### Таблица: `email_verification_codes`

```sql
CREATE TABLE email_verification_codes (
  id UUID PRIMARY KEY,
  email VARCHAR(255) NOT NULL,
  code VARCHAR(6) NOT NULL,
  expires_at TIMESTAMP NOT NULL,
  is_used BOOLEAN DEFAULT FALSE,
  type VARCHAR(20) DEFAULT 'login', -- 'login' или 'registration'
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_email_code ON email_verification_codes(email, code);
```

## 🧪 Тестирование

### Тест 1: Регистрация с 2FA

1. Откройте http://localhost:3000/auth/signin
2. Переключитесь на "Регистрация"
3. Введите:
   - Email: `test@example.com`
   - Имя: `Тест`
   - Пароль: `MySecure123!`
4. Нажмите "Зарегистрироваться"
5. **Ожидаемый результат:** Переход на страницу ввода кода
6. Проверьте терминал - там будет 6-значный код
7. Введите код
8. **Ожидаемый результат:** Вход в профиль

### Тест 2: Вход с 2FA

1. Выйдите из аккаунта
2. Откройте http://localhost:3000/auth/signin
3. Введите email и пароль
4. Нажмите "Войти"
5. **Ожидаемый результат:** Переход на страницу ввода кода
6. Проверьте терминал - там будет новый код
7. Введите код
8. **Ожидаемый результат:** Вход в профиль

### Тест 3: Неправильный код

1. Войдите (получите код)
2. Введите неправильный код: `000000`
3. **Ожидаемый результат:** "Неверный код"
4. Поля очищаются, фокус на первом поле

### Тест 4: Истекший код

1. Войдите (получите код)
2. Подождите 10 минут (или измените `expires_at` в БД)
3. Введите код
4. **Ожидаемый результат:** "Код истек. Запросите новый код"

### Тест 5: Повторная отправка кода

1. Войдите (получите код)
2. Нажмите "Отправить повторно"
3. **Ожидаемый результат:** Кнопка неактивна 60 секунд
4. Проверьте терминал - там будет новый код
5. Старый код больше не работает

### Тест 6: Вставка кода

1. Войдите (получите код из терминала)
2. Скопируйте код: `123456`
3. Нажмите Ctrl+V в любом поле
4. **Ожидаемый результат:** Все поля заполняются, автоматическая отправка

В файле `lib/verification-code.ts`:

```typescript
export function generateVerificationCode(): string {
  // Для 4-значного кода:
  return Math.floor(1000 + Math.random() * 9000).toString();
  
  // Для 8-значного кода:
  return Math.floor(10000000 + Math.random() * 90000000).toString();
}
```

## 📝 API Endpoints

### POST /api/auth/send-code
Отправляет код верификации на email

**Request:**
```json
{
  "email": "user@example.com",
  "password": "MySecure123!",
  "type": "login" // или "registration"
}
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Код отправлен на ваш email"
}
```

**Response (Error):**
```json
{
  "error": "Неверный пароль. Осталось попыток: 3"
}
```

### POST /api/auth/verify-code
Проверяет код верификации

**Request:**
```json
{
  "email": "user@example.com",
  "code": "123456",
  "type": "login"
}
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Код подтвержден",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "User Name",
    "role": "user"
  }
}
```

**Response (Error):**
```json
{
  "error": "Неверный код"
}
// или
{
  "error": "Код истек. Запросите новый код"
}
```
