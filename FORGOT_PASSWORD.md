# 🔑 Функция "Забыли пароль?"

## ✅ Реализовано

Полный процесс восстановления пароля с 6-значным кодом:

```
1. Пользователь нажимает "Забыли пароль?"
   ↓
2. Вводит email
   ↓
3. Получает 6-значный код на email
   ↓
4. Вводит код на странице
   ↓
5. Создает новый пароль (+ повтор)
   ↓
6. Пароль сброшен → вход в систему
```

---

## 📋 Процесс восстановления

### Шаг 1: Страница входа
**URL:** `/auth/signin`

- Кнопка "Забыли пароль?" под формой входа
- Видна только в режиме входа (не при регистрации)

### Шаг 2: Ввод email
**URL:** `/auth/forgot-password`

Пользователь:
- Вводит email
- Нажимает "Отправить код"

Система:
- Проверяет существование пользователя
- Генерирует 6-значный код
- Отправляет код на email
- Переходит на страницу ввода кода

**Безопасность:** Всегда показывает успех, даже если email не найден (защита от перечисления пользователей)

### Шаг 3: Ввод кода
**URL:** `/auth/reset-password?email=...`

Пользователь:
- Видит 6 полей для ввода кода
- Может вставить код (Ctrl+V)
- Может запросить новый код (таймер 60 сек)

Система:
- Проверяет код
- Проверяет срок действия (10 минут)
- При правильном коде → переход к созданию пароля

### Шаг 4: Создание нового пароля
**URL:** `/auth/reset-password?email=...` (тот же, но другой шаг)

Пользователь:
- Вводит новый пароль
- Повторяет пароль
- Может показать/скрыть пароль (иконка глаза)

Система:
- Проверяет совпадение паролей
- Валидирует безопасность пароля
- Сохраняет новый пароль
- Сбрасывает блокировку аккаунта
- Переходит на страницу входа

---

## 🔒 Безопасность

### Защита кода:
- ✅ **Срок действия:** 10 минут
- ✅ **Одноразовый:** После использования код становится недействительным
- ✅ **Тип:** `reset_password` (отдельный от login/registration)
- ✅ **Удаление старых:** При запросе нового кода старые удаляются

### Защита от перечисления:
- ✅ Всегда возвращает успех, даже если email не существует
- ✅ Не раскрывает информацию о существовании аккаунта

### Валидация пароля:
- ✅ Минимум 8 символов
- ✅ Заглавные и строчные буквы
- ✅ Минимум 1 цифра
- ✅ Минимум 1 спецсимвол

### Сброс блокировки:
- ✅ При успешном сбросе пароля счетчик попыток входа сбрасывается
- ✅ Блокировка аккаунта снимается

## 📊 API Endpoints

### POST /api/auth/forgot-password
Отправляет код для сброса пароля

**Request:**
```json
{
  "email": "user@example.com"
}
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Код для сброса пароля отправлен на ваш email"
}
```

### POST /api/auth/reset-password
Сбрасывает пароль после проверки кода

**Request:**
```json
{
  "email": "user@example.com",
  "code": "123456",
  "newPassword": "NewPass123!"
}
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Пароль успешно изменен"
}
```

**Response (Error - слабый пароль):**
```json
{
  "error": "Пароль не соответствует требованиям безопасности",
  "details": [
    "Пароль должен содержать минимум 1 заглавную букву",
    "Пароль должен содержать минимум 1 спецсимвол"
  ]
}
```
